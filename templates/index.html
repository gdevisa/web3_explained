Web3 Explained and Examined

What is it and how is it going to change the Internet?

..but first, let's take a look at the history



Web 1.0

(1991-2004)

Refers to the first stage of the World Wide Web

  • Read only - static pages user can't change/interact with
  • Content provided from an ISP-hosted web server's file system or from one's own server
  • Pages built using Server Side Includes or Common Gateway Interface instead of a dynamic programming language

Web 2.0

(2004-Now)

The second stage in the evolution - the "Social" Web

  • Read AND Write webpages - information flows both ways between the user and the site, user generated content as an essential part
  • Dynamic pages that are responsive to user input
  • Content is rather provided from a relational database management system or "the cloud"
  • Pages built using dynamic programming languages and frameworks like JavaScript, AJAX, Python, PHP, etc.
  • Software as a Service - websites developed APIs to allow self-usage, such as by a web/software application

So, What's Next? The New Internet?

Decentralized, secure, private. That is what some Web3 technologies promise...

But it's not that simple

The whole evolution has been a gradual process with some general time periods referred to as "1.0" or "2.0"

Right now, we are in the middle of another paradigm/trend shift

It's pretty hard to define something which is not a thing yet, or not even a thing but rather a bundle of technologies used by different websites because there isn't one central "mothership" with its software running everything

"The official tale" - Web 3.0

as per the World Wide Web Consortium (W3C)

They call it the "Semantic" Web. Basically, altering it into a database, the whole point of which is to organize the information in a more machine-friendly, machine-readable way.

This way the Artificial Intelligence would be able to leverage the user data and semantic metadata, and provide faster and more relevant results. Something that we already see implemented widely on a website/platform level

Content is accessible by multiple applications, for any device connected, and the services can be used anywhere

Also, can be characterized by use of 3D graphics, AR/VR, other ways of representing or transmitting information to/by users like livestreams

By and large, already what we have today

"The community tale" - Web3

Probably the real reason why you're here

At its core, Web3 is a rebranding of crypto and blockchain, the technology based around a worldwide network of computers that talk to each other and validate and record transactions without human intervention or centralized oversight

The idea is to apply blockchain technology in different ways to make the Internet more secure, private, "give the power back to the users", etc.

The advocates of Web3 look at the Internet from a bit of a different angle, focusing more on the power relationship between the companies providing services with their websites and users/consumers

As explained on the Ethereum.org, "Web2 refers to the version of the internet most of us know today. An internet dominated by companies that provide services in exchange for your personal data. Web3, in the context of Ethereum, refers to decentralized apps that run on the blockchain. These are apps that allow anyone to participate without monetising their personal data."

Ethereum (blockchain) network is very important here because it's the one leading charge in the development of Web3. For example, the NFTs, which allow someone to own digital artworks, run on the Ethereum network utilizing its Smart Contracts feature. For now, this is the biggest actual manifestation of Web3 on the Internet today.

But the proponents of Web3 are talking about bigger things than just selling and collecting images, right? Right

Therefore, dApps - decentralized applications, which run on the Ethereum or analogue network (blockchain)

Basically, an alternative to the existing architecture of websites/apps where a given company has a server which holds all the data and runs all the back-end code for it to function, and instead running all those server side processes on the distributed decentralized Ethereum (or other) network

Some believers in Web3 even think that, in the future, it would be possible to run tech companies using the same underlying mechanisms and technologies by having DAOs - decentralized autonomous organizations

OK, I see where you're going

Let's start bringing it all together

Web2 vs Web3

Some Practical Comparison

Web2 Web3
Social media companies can censor any account or post Web3 content would be uncensorable because control is decentralized
Payment service may decide to not allow payments for certain types of work Web3 payment apps require no personal data and can't prevent payments
Servers for gig-economy apps could go down and affect worker income Web3 servers can't go down – they use Ethereum (or other network), a decentralized network of thousands of computers as their back end

Web3 Limitations

It's not that easy

  • Scalability – transactions are slower on Web3 because they're decentralized. Changes to state, like a payment, need to be processed by a miner and propagated throughout the network
  • Inclusion - not everyone can or is willing to run a node of the blockchain on their device (especially mobile)
  • UX – interacting with Web3 applications can require extra steps, software, and education. This can be a hurdle to adoption
  • Accessibility – the lack of integration in modern web browsers makes web3 less accessible to most users
  • Incentive - if no one owns anything, then without financial motivation there will be fewer people interested in development. A possible solution to that is minting tokens for developers/contributors, but how that would work in a particular case is not clear
  • Cost – most successful dApps put very small portions of their code and you can't write much data on the blockchain as it's expensive

Looking at everything that we have at hand right now, I want to argue that Web3 as imagined by most of its proponents is not going to happen, at least not in the same way or scale. For the most part, what Web3 boosters talk about is theoretical and does not have good precedents yet, while there already are serious issues undermining the future success.


Decentralization. The main goal of Web3 in many regards. They say everyone is going to be a peer in the network, “share the burden”, therefore, decentralization, equality, etc. But that is not entirely feasible. Becoming an actual peer/node in the network requires for a “client”/server of Ethereum (or other network) to run on your machine, or in other words for you to be connected to the blockchain to modify or render its state, which a) isn’t the most user friendly and easy thing to set up on your machine, b) excludes mobile devices and you can’t just run it in a desktop web browser, and c) a lot of people are fine with a company running everything on their servers and providing services (remember, that was part of the switch from Web 1.0 to 2.0). Therefore, a significant chunk of the people aren’t gonna be full participants of the network, which undermines the whole idea of decentralization. It would be more appropriate to call it a “redistribution of the Internet” into the hands of actual peers. Wait, but then how are we able to open Web3 websites that already exist simply in our web browser without installing anything?


Centralization. The way it is actually done at the moment is that you interact with a node that’s running remotely on a server. But the whole point was to get rid of the servers because they create centralization, right? Right. Moreover, almost all dApps use either Infura or Alchemy for development and interaction with the blockchain. They can’t somehow tamper with a blockchain because it’s all publicly available and distributed, but all the read requests are still going through them and the outputs these websites’ APIs give about the state of the chain aren’t verified anyhow so all the clients are just trusting their response. We see the same with centralized crypto exchanges. This can potentially undermine privacy and security of the whole system. Some see it as a necessary compromise, and think that in the event if the companies misbehave somehow the users can just leave and go elsewhere, but that doesn’t sound like a very practical approach to me. Although there are chains which promise easier, direct access which would solve this issue, they are under the radar for most people and aren’t being hyped resulting in further building of Web3 on the wrong foundation.


Cost. Another big issue which makes the previous one even worse. For example, as a cryptographer and a former CEO of Signal Moxie Marlinspike found out, if you own an NFT on OpenSea and it’s removed from there, then you won’t be able to find it in your crypto wallet precisely because there happens to be a chain of API calls from the wallet app to Etherscan to Infura and then routing back to call OpenSea API to see the NFT. In this case, the main issue is in that last step and the fact that an NFT’s data isn’t stored on the chain, instead it’s just that API link. The reason is it would be extremely costly to store that much data on the blockchain, which is another aforementioned in the overview limiting factor for Web3.


So, it’s not that “bright and shiny” as some Web3 idealists might want to think. There are going to have to be a lot of compromises made. Looking back at the history of the Internet and tech sector in general, we see cycles of centralization and decentralization. For instance, after WW2 everything was very centralized in the government's hands and after that at some point came hobbyists building their own computers at home, who then started creating their own companies. A similar but reverse thing happened with software, at first you had to build an OS by yourself, then came Microsoft and basically took over most of the market. The Internet itself, Web 1.0, was open for all, decentralized as Web3 enthusiasts would say, and then came Web 2.0 which got centralized by the big tech. Now, they advocate, it’s time to decentralize it again with Web3. So, they’re following the cycle and I agree that there are issues with Web2 as it is now and that would be a natural course of events, but I think some people are clouded by the hype and their vision of “completely decentralized, owned by the people, trustless Internet”, is unrealistic. The government is still heavily involved in the tech sector, not to the same extent as in the 1960s, of course, but it didn’t just disappear as a thing, and in fact they work a lot together with private companies. Most of the computers come with either Windows or MacOS, but anyone is still free to find other free software online or build their own. Again, a lot of people are just happy to consume what exists. I believe a similar thing will happen with Web3 in the end. Not all the websites are gonna move to Web3 standards, not all of them need that, and the ones that will are going to have to work differently than what we see now in order to deliver on the promise of true decentralization, privacy, and security.
